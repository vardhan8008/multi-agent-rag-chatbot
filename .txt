# Conversational RAG for Developers

This repository contains a compact, practical Retrieval-Augmented Generation
(RAG) assistant you can run locally or in Docker. It's designed for multi-turn
conversations about uploaded documents: you index files, ask questions, and
keep context across follow-ups via per-session memory.

Architecture (simple):

Streamlit UI  ──HTTP──▶  FastAPI  ──▶  LangGraph (analyze → retrieve → generate)
                                          │            └─▶ Chroma (vector store)
                                          └─▶ MemorySaver (per-session memory)

Key points:
- End-to-end pipeline: ingest → chunk → embed → store → retrieve → answer
- LangGraph coordinates routing, retrieval and response generation
- Local embeddings (`all-MiniLM-L6-v2`) and a persistent Chroma DB
- FastAPI provides upload, chat, session and history endpoints
- Streamlit gives a minimal chat UI with source citations

Technology choices
- LLM: Groq `llama-3.1-8b-instant` (requires a free Groq API key)
- Embeddings: `all-MiniLM-L6-v2` (HuggingFace, runs locally)
- Vector store: Chroma (persistent, file-backed)
- Orchestration: LangGraph

Quick start (local)

1. Get a Groq API key: https://console.groq.com/keys

2. Create and activate a virtualenv, then install dependencies:

```powershell
python -m venv .venv
.venv\\Scripts\\activate
pip install -r requirements.txt
```

3. Configure environment variables (Windows):

```powershell
copy .env.example .env
# Edit .env and set GROQ_API_KEY
```

4. Start the API (development):

```powershell
uvicorn app.main:app --reload
```

Open the OpenAPI docs at http://localhost:8000/docs

5. Start the UI in a second terminal:

```powershell
streamlit run ui/streamlit_app.py
```

By default the UI runs at http://localhost:8501. Note: the embedding model
downloads on the first run (~90 MB).

Quick start (Docker)

```powershell
copy .env.example .env
# add your GROQ_API_KEY to .env
docker compose up --build
```

Services:
- API → http://localhost:8000/docs
- UI → http://localhost:8501

Usage notes
- Upload `.pdf`, `.txt` or `.md` via the Streamlit sidebar and click
  "Index document" to add content to Chroma.
- Ask a question in the chat. Follow-ups keep context because LangGraph
  rewrites pronoun-based queries into standalone prompts.
- Use "New conversation" to clear session memory and start fresh.

Project layout

```

├── app/
│   ├── main.py            # FastAPI app + routes
│   ├── config.py          # environment-driven settings
│   ├── schemas.py         # request/response models
│   ├── memory.py          # session registry
│   └── rag/
│       ├── loader.py      # document reading + chunking
│       ├── llm.py         # Groq LLM wrapper + embeddings
│       ├── vectorstore.py # Chroma wrapper
│       └── graph.py       # LangGraph orchestration and agents
├── ui/
│   └── streamlit_app.py   # minimal chat UI
├── docs/                  # architecture, API, samples
├── data/                  # uploads + chroma persistence (gitignored)
├── Dockerfile
├── docker-compose.yml
├── requirements.txt

```
